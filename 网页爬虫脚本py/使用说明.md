# 高考教育院校专业组爬虫 — 使用说明

本目录用于从 [高考教育网](https://www.gaokao.cn) 批量抓取院校「招生计划 / 专业组 / 选科 / 包含专业」数据，并汇总到 Excel；同时支持将爬取到的院校 ID 合并到教育部「普通高校」名单。

**浏览器自动化已改用 Playwright**，无需单独下载 ChromeDriver。

---

## 一、目录文件清单

### Python 脚本（4 个）

| 文件 | 作用 | 执行顺序 |
|------|------|----------|
| `抓取大学名字.py` | 按院校 ID 批量访问 `gaokao.cn/school/{id}`，建立「院校名称 ↔ 代码」映射表 | **爬取第 1 步** |
| `2-合并院校id到普通高校.py` | 将映射表中的 ID 合并进教育部普通高校名单 | **数据第 2 步**（爬取完成后执行） |
| `通过id爬取大学的专业组.py` | 按 **院校代码（ID）** 进入招生计划页，抓取专业组明细 | **爬取第 2 步**（主流程） |
| `通过院校名称爬取大学专业组.py` | 按 **院校名称** 搜索并抓取专业组明细 | **爬取第 3 步**（补爬 / 兜底） |

### 辅助脚本

| 文件 | 作用 |
|------|------|
| `playwright_utils.py` | 浏览器启动、登录等待等共用逻辑 |

### JavaScript 脚本（2 个）

| 文件 | 作用 |
|------|------|
| `控制台-中国教育-学校专业组.js` | **核心抓取逻辑**：在院校详情页自动切换省份（江西）、科类、专业组，提取表格并格式化为 Excel 数据 |
| `抓取大学名字.js` | 浏览器控制台版（有页面跳转缺陷，一般不用） |

### 数据文件

| 文件 | 类型 | 列结构 | 说明 |
|------|------|--------|------|
| `高考教育院校id_map_表.csv` | **爬取主产物** | `院校名称`、`代码`、`状态`、`错误信息`、`日期` | `抓取大学名字.py` 实时写入，**断点续爬依据** |
| `高考教育院校id_map_表.xlsx` | 备份 / 旧数据 | 同上 | 无 csv 时启动可自动转为 csv |
| `普通高校.xls` | **输入** | `序号`、`学校名称`、`学校标识码`、`主管部门`、`所在地`、`办学层次`、`备注` | 教育部普通高校名单（只读，不修改原文件） |
| `普通高校_带id.xlsx` | **合并输出** | 原表列 + `id` | 匹配写 id；校区 / 未匹配院校在末尾新增行 |
| `还需要爬取的院校.xlsx` | **输入** | `院校名称`、`状态` | 待补爬名单 |
| `院校招生专业组专业明细.xlsx` | **最终输出** | `院校名称`、`专业组`、`选科`、`包含专业` | 专业组数据汇总 |

## 二、整体流程

### A. 爬取流程

```text
┌─────────────────────────────────────────────────────────────────┐
│  爬取第 1 步  抓取大学名字.py                                      │
│  产出：高考教育院校id_map_表.csv（1 ~ 4000）                        │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  爬取第 2 步  通过id爬取大学的专业组.py                             │
│  输入：高考教育院校id_map_表.xlsx                                  │
│  产出：院校招生专业组专业明细.xlsx（追加）                          │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
                    仍有失败 / 未收录的院校？
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  手动编辑：还需要爬取的院校.xlsx                                   │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  爬取第 3 步  通过院校名称爬取大学专业组.py                          │
│  产出：院校招生专业组专业明细.xlsx（继续追加）                       │
└─────────────────────────────────────────────────────────────────┘
```

### B. 数据合并流程（独立）

```text
┌─────────────────────────────────────────────────────────────────┐
│  数据第 2 步  2-合并院校id到普通高校.py                            │
│  输入：高考教育院校id_map_表.csv + 普通高校.xls                    │
│  产出：普通高校_带id.xlsx                                          │
└─────────────────────────────────────────────────────────────────┘
```

> 合并在爬取映射表完成后执行即可，与专业组爬取无先后强依赖。

## 三、各步详细说明

### 爬取第 1 步：`抓取大学名字.py`

**访问范围：** `https://www.gaokao.cn/school/1` ~ `school/4000`

**页面判断逻辑：**
1. 等待 `.main-nav_logo2__bmYaw` 出现（页面加载完成，超时 3s）
2. URL 为首页 → **无效**（ID 无院校），立刻扫下一个，不额外等待
3. 有校名 → **成功**
4. 目标页无校名 → **无效**
5. 其他异常 → **失败**（下次启动重试）

**状态说明：**

| 状态 | 含义 | 下次启动 |
|------|------|----------|
| 成功 | 抓到院校名称 | `日期` 在 **半年内** 跳过，超过半年重拉 |
| 无效 | 跳首页 / 无校名，确认 ID 无院校 | `日期` 在 **半年内** 跳过，超过半年重拉 |
| 失败 | 加载超时、跳转异常等 | **重试** |

**断点续爬：** 以 `高考教育院校id_map_表.csv` 为准；`成功` / `无效` 且 `日期` 距今不超过 6 个月才跳过，更早的数据会自动重新抓取。

**登录：** 启动后先在浏览器登录，终端按回车继续；登录态保存在 `.playwright_gaokao_profile/`。

---

### 数据第 2 步：`2-合并院校id到普通高校.py`

**输入：**
- `高考教育院校id_map_表.csv`（仅处理 `状态=成功` 且有院校名的行）
- `普通高校.xls`（教育部名单，原文件不修改）

**合并规则：**

| 情况 | 处理 |
|------|------|
| 院校名称含 **「校区」** | 在普通高校表**新增一行**（学校名称 + `id`） |
| 去空格后名称与普通高校 **完全相等** | 若无 `id` 列则新增，写入高考教育 `代码` 到该行 `id` |
| 在普通高校中 **找不到** | 在 `普通高校_带id.xlsx` **末尾新增一行**（学校名称 + `id`） |

**名称比对：** 去除半角 / 全角空格后精确匹配。

**运行：**

```bash
python 2-合并院校id到普通高校.py
```

**产出示例：**
- `普通高校_带id.xlsx` — 如「北京大学」行 `id=31`；未匹配院校（如「上海交通大学医学院」）追加在表末

---

### 爬取第 2 步：`通过id爬取大学的专业组.py`

1. 读取 `高考教育院校id_map_表.xlsx`
2. 只处理 `状态 != "成功"` 的行
3. 打开 `https://www.gaokao.cn/school/{代码}/provinceline`
4. 注入 `控制台-中国教育-学校专业组.js` 抓取
5. 追加到 `院校招生专业组专业明细.xlsx`

---

### 爬取第 3 步：`通过院校名称爬取大学专业组.py`

1. 读取 `还需要爬取的院校.xlsx`
2. 通过 `headSearch` 搜索并点击匹配院校
3. 同样注入 JS 抓取，追加专业组明细

## 四、环境准备

```bash
cd 网页爬虫脚本py
pip install -r requirements.txt
playwright install chromium   # 仅爬取脚本需要
```

| 包 | 用途 |
|----|------|
| `playwright` | 浏览器自动化 |
| `pandas` | 读写 Excel / CSV |
| `openpyxl` | 读写 `.xlsx` |
| `xlrd` | 读取 `.xls`（普通高校名单） |

## 五、如何执行

```bash
cd 网页爬虫脚本py

# 爬取第 1 步：建立 ID 映射
python 抓取大学名字.py

# 数据第 2 步：合并到普通高校名单（映射表完成后）
python 2-合并院校id到普通高校.py

# 爬取第 2 步：按 ID 批量抓专业组
python 通过id爬取大学的专业组.py

# 爬取第 3 步：按名称补爬
python 通过院校名称爬取大学专业组.py
```

**爬取脚本运行期间：**
- 有头模式，弹出 Chromium 窗口
- 启动后先登录，回车继续
- 可随时 Ctrl+C 中断，失败行下次重试

## 六、输入文件准备

### `高考教育院校id_map_表.csv`

| 列名 | 说明 |
|------|------|
| 院校名称 | 院校全称；无效 ID 为空 |
| 代码 | 高考教育 URL 中的数字 ID |
| 状态 | `成功` / `无效` / `失败` |
| 错误信息 | 失败或无效原因 |
| 日期 | 最近一次抓取日期，格式 `YYYY-MM-DD`（用于判断是否超过半年需重拉） |

### `普通高校.xls`

教育部公布名单，列：`序号`、`学校名称`、`学校标识码`、`主管部门`、`所在地`、`办学层次`、`备注`。

### `还需要爬取的院校.xlsx`

| 列名 | 说明 |
|------|------|
| 院校名称 | 与网站搜索一致的完整校名 |
| 状态 | 留空待爬；`成功` 跳过；`失败` 重试 |

## 八、常见问题

| 问题 | 处理建议 |
|------|----------|
| `playwright install` 下载慢 | 多试几次，只需安装一次 |
| 启动后没给登录时间 | 等终端提示，浏览器登录后再按回车 |
| 想重新登录 | 删除 `.playwright_gaokao_profile` 后重跑 |
| ID 2、3、4 显示无效 | 正常，低 ID 区间多数无院校 |
| 合并后表末新增很多行 | 多为医学院、职院、校区等名称与教育部名单不一致，属正常 |
| 想重新爬某校 | 将该行 `状态` 改为 `失败` 后重跑对应脚本 |
| 想重新合并 | 直接重跑 `2-合并院校id到普通高校.py`（会覆盖输出文件） |
